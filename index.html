1 · Instalar bridge script

Corre este script en tu PC conectada al mismo WiFi que la cámara QC5. Solo necesitas Node.js instalado.

npm install -g ffmpeg-static git clone github.com/tu-usuario/vecindariocam cd vecindariocam/bridge cp .env.example .env # Edita .env con tu URL Supabase node bridge.js

2 · Encontrar IP de la cámara

La cámara QC5 generalmente expone RTSP en el puerto 554. Primero encuentra su IP en tu router.

# En terminal (mismo WiFi): # Mac/Linux: arp -a | grep -i "d4:a3" # Probar stream con VLC: rtsp://admin:@192.168.X.X:554/stream # URL común QC5: rtsp://IP:554/onvif1

3 · Variables de entorno

El bridge necesita conectarse a tu proyecto Supabase. Configura estas variables.

SUPABASE_URL=https://xxx.supabase.co SUPABASE_SERVICE_KEY=eyJ... RTSP_URL=rtsp://admin:@192.168.1.X:554/onvif1 CLIP_DURATION=120 CAMERA_ID=qc5-estacionamiento RETENTION_HOURS=72

4 · Tabla Supabase

Ejecuta este SQL en el editor de Supabase para crear la tabla de grabaciones.

create table recordings ( id uuid default gen_random_uuid() primary key, camera_id text not null, started_at timestamptz not null, duration_sec int, storage_path text not null, has_motion bool default false, created_at timestamptz default now() ); -- RLS: solo usuarios auth pueden leer alter table recordings enable row level security; create policy "vecinos read" on recordings for select using (auth.role() = 'authenticated');

Instrucciones Supabase Auth — vecinos

Para agregar vecinos autorizados, ve a tu panel Supabase → Authentication → Users → Invite user. Solo necesitan su correo y crearán su contraseña. No necesitas código adicional, Supabase maneja el registro por invitación.

  • Ir a supabase.com → tu proyecto → Authentication
  • Pestaña "Users" → botón "Invite user"
  • Ingresar el correo del vecino y enviar invitación
  • El vecino recibirá un email para establecer su contraseña
  • Ya puede acceder a VecindarioCam con sus credenciales
